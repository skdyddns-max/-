# 능력 재고 (Capability Inventory)

[[메타 1 — 논리 레벨 분석]]에서 가장 얇다고 진단된 3층(능력)을 채우는 노트. "현장의 문제를 AI 제품으로 해결하는 사람"이 되기 위한 능력을 **보유 / 부족**으로 나눈다. 보유 능력은 추측이 아니라 **이 저장소의 산출물을 증거로** 적었다.

## A. 임상 능력 (17년의 자산 — 경쟁자가 못 따라오는 것)

| 능력 | 증거 | 제품에 어떻게 실리나 |
|---|---|---|
| 아동 언어 진단·중재 설계 | 17년 임상 | 워크시트 분석기의 "임상적 해석" 로직 |
| AAC 평가·적용 | H·J 아동 맞춤 AAC 진행 중 | AAC 자동 생성기의 설정 항목 설계 |
| 화용언어 중재 | 화용 스튜디오의 시나리오·퀴즈 구조 | 대본 생성 프롬프트의 임상 규칙 |
| 보호자 상담 | 17년 임상 | 보호자용 리포트의 언어 수위 조절 |
| 단일대상연구 설계 (기초선-중재 비교) | 임상 훈련 | **원칙 7(효과는 기록으로)의 측정 방법론** — 개발자 출신은 못 하는 것 |

## B. 제작 능력 — 보유 (저장소가 증거)

| 능력 | 증거 |
|---|---|
| 단일 파일 웹앱 설계 (빌드·백엔드 없음) | `studio.html`, `tonemaker.html`, `product.html` |
| LLM API 연동 + 프롬프트 설계 | Claude/Gemini API, 프롬프트 복사 폴백 모드 |
| 생성형 미디어 파이프라인 | Veo 영상, Gemini TTS 더빙, 이미지 생성(감정 카드) |
| 로컬 우선 데이터 설계 | localStorage-only, CSV 내보내기, EXIF 제거 |
| PWA / 오프라인 배포 | 홈 화면 설치, 독립 실행형 치료 플레이어 내보내기 |
| 기록·측정 장치 내장 | 퀴즈 정답률·시도 횟수 자동 저장 (원칙 7의 싹) |

## C. 제작 능력 — 부족 (TalkBloom으로 가는 길의 병목)

| 부족한 능력 | 언제 필요해지나 | 채우는 방법 (제안) |
|---|---|---|
| 백엔드·계정·DB | 파이프라인 5단계(플랫폼) — 치료사 10명이 각자 데이터를 가지려면 | 탐색 트랙 프로젝트 1개를 일부러 백엔드 포함으로 |
| 결제·구독 | [[메타 4 — 빈칸과 질문]] 빈칸 1(지속가능성) 실험 시점 | 기존 결제 서비스 연동으로 최소 실험 |
| 아동 개인정보 법규 (개인정보보호법·의료정보) | 데이터가 기기 밖으로 나가는 첫 순간 | 전문가 자문 1회 + 원칙 6 체크리스트화 |
| 유통·도달 (치료사 커뮤니티) | 1년 이정표 "치료사 10명" | 언어재활사 커뮤니티·학회에 사용 사례 공유 |

## 사용법

- 새 프로젝트 착수 전: C열에 걸리는 게 있으면 **범위를 줄이거나, 그 능력을 익히는 것 자체를 프로젝트 목표에 포함**한다.
- 분기 점검([[템플릿 — 분기 세계관 점검]]) 때 B/C 사이를 옮긴다 — C가 B로 이동한 기록이 곧 성장 로그다.
