# 미래 프로젝트 — 만들고 싶은 것

v0.1의 나열식 8개를 **사용자별 × 단계별**로 재배열한 것. (원문 순서: 치료일지 생성기, 보호자 상담 리포트, 언어평가 분석기, AAC 자동 생성기, 자료 생성 플랫폼, 업무 자동화 도구, AI 치료 코치, TalkBloom)

## 사용자별 재배열

| 직접 사용자 | 도구 | 최종 수혜자에게 닿는 경로 |
|---|---|---|
| 치료사 | AI 치료일지 생성기 · AI 언어평가 분석기 · 언어치료 자료 생성 플랫폼 · 치료사 업무 자동화 도구 | 문서 시간 ↓ → 아이와의 시간 ↑ |
| 보호자 | AI 보호자 상담 리포트 · AI 치료 코치 | 가정 내 연습의 질 ↑ → 치료실 밖 일반화 |
| 아이 | AAC 자동 생성기 | 의사소통 수단 직접 제공 |
| 전체 | **TalkBloom 통합 플랫폼** | 위 전부의 지붕 |

이 표가 드러내는 것: 8개 중 아이가 직접 쓰는 것은 1개뿐이다. 모순이 아니라 **가치 전달 사슬**이다 — 치료사와 보호자는 아이에게 닿는 통로다. 단, v0.2에서 이 사슬을 명시해야 한다 → [[메타 3 — 긴장과 모순]]의 긴장 3.

새 프로젝트를 시작하기 전에 반드시 통과할 문:

2. [[원칙 3 — 한 아이를 위해 만들고 많은 아이에게 확장한다]] — 첫 사용자 한 명이 정해져 있는가?
3. [[메타 3 — 긴장과 모순]]의 긴장 2 점검 — 기존 제품을 개선하는 게 더 낫지 않은가?
