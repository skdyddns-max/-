# 미래 프로젝트 — 만들고 싶은 것

v0.1의 나열식 8개를 **사용자별 × 단계별**로 재배열한 것. (원문 순서: 치료일지 생성기, 보호자 상담 리포트, 언어평가 분석기, AAC 자동 생성기, 자료 생성 플랫폼, 업무 자동화 도구, AI 치료 코치, TalkBloom)

## 사용자별 재배열

| 직접 사용자 | 도구 | 최종 수혜자에게 닿는 경로 |
|---|---|---|
| 치료사 | AI 치료일지 생성기 · AI 언어평가 분석기 · 언어치료 자료 생성 플랫폼 · 치료사 업무 자동화 도구 | 문서 시간 ↓ → 아이와의 시간 ↑ |
| 보호자 | AI 보호자 상담 리포트 · AI 치료 코치 | 가정 내 연습의 질 ↑ → 치료실 밖 일반화 |
| 아이 | AAC 자동 생성기 | 의사소통 수단 직접 제공 |
| 전체 | **TalkBloom 통합 플랫폼** | 위 전부의 지붕 |

이 표가 드러내는 것: 8개 중 아이가 직접 쓰는 것은 1개뿐이다. 모순이 아니라 **가치 전달 사슬**이다 — 치료사와 보호자는 아이에게 닿는 통로다. 단, v0.2에서 이 사슬을 명시해야 한다 → [[메타 3 — 긴장과 모순]]의 긴장 3.

## 제품별 한 줄 스펙 (입력 → 출력 → 첫 사용자)

| 제품 | 입력 | 출력 | 첫 사용자 (N=1) | 선행 조건 |
|---|---|---|---|---|
| AI 치료일지 생성기 | 세션 메모(음성/타이핑) + 치료 목표 | 기관 양식에 맞는 치료일지 초안 | 나 (내 일지부터) | 없음 — 가장 빨리 착수 가능 |
| AI 보호자 상담 리포트 | 워크시트 분석 결과 + 세션 기록 | 보호자 언어로 번역된 월간 리포트 + 가정 활동 제안 | 내 케이스 보호자 1명 | 워크시트 분석기 데이터 축적 |
| AI 언어평가 분석기 | 공식 검사 결과(원점수) | 프로파일 해석 + 목표 후보 + 보고서 초안 | 나 | 검사 저작권 범위 확인 필요 |
| AAC 자동 생성기 | 아동 정보(어휘 수준·운동 능력·관심사) | 맞춤 AAC 보드 세트 | 세 번째 AAC 대상 아동 | **H·J 두 사례의 차이점 목록** ([[현재 프로젝트]]) |
| 언어치료 자료 생성 플랫폼 | 치료 목표 + 아동 특성 | 워크시트·그림카드·활동지 | 나 → 동료 치료사 1명 | 탐색 트랙의 이미지 파이프라인 회수 |
| 치료사 업무 자동화 도구 | (위 도구들의 묶음) | — | — | 단독 제품이 아니라 위 도구들의 총칭일 수 있음 — 분리 재검토 |
| AI 치료 코치 | 치료사의 세션 계획 | 중재 전략 제안·피드백 | 저연차 치료사 1명 | 효과 데이터 축적 (근거 없는 코치는 원칙 4 위반) |
| TalkBloom | 위 전부 | 통합 플랫폼 | — | 유료 실험 + 백엔드 능력 ([[능력 재고]] C열) |

**추천 착수 순서**: 치료일지 생성기(선행 조건 없음, 매주 내가 겪는 문제) → 보호자 리포트(분석기와 연결) → AAC 자동 생성기(세 번째 사례가 생길 때). "치료사 업무 자동화 도구"는 별도 제품이 아니라 카테고리명일 가능성이 높으니 v0.2 목록에서 정리 검토.

## 착수 순서의 원칙

새 프로젝트를 시작하기 전에 반드시 통과할 문:

1. [[원칙 1 — 모든 시작은 현장에서 나온다]] — 지금 내 치료실에서 아픈 문제인가?
2. [[원칙 3 — 한 아이를 위해 만들고 많은 아이에게 확장한다]] — 첫 사용자 한 명이 정해져 있는가?
3. [[메타 3 — 긴장과 모순]]의 긴장 2 점검 — 기존 제품을 개선하는 게 더 낫지 않은가?

실제 착수 시에는 [[템플릿 — 새 프로젝트 킥오프]]의 5문 게이트를 통과할 것.
