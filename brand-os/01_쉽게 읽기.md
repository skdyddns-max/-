# 📖 쉽게 읽기 — 이 볼트가 뭔지 5분 안에 이해하기

> 어려운 말이 나오면 이 노트로 돌아오세요. 여기 없는 말이 볼트에 있다면 그건 잘못 쓴 겁니다.

## 1. 이 폴더는 뭔가요?

**내 브랜드의 설계도**입니다. "나는 누구고, 왜 이 일을 하고, 뭘 만들고, 뭘 안 만들 것인가"를 적어 둔 문서 모음이에요.

한 파일에 다 쓰지 않고 여러 개의 짧은 노트로 쪼개서 서로 연결해 뒀습니다. 이유는 하나예요 — **생각은 부분마다 바뀌는 속도가 다르기 때문**입니다. "왜 이 일을 하는가"는 10년 가지만, "지금 뭘 만드는가"는 석 달마다 바뀝니다. 한 파일이면 매번 전체를 다시 써야 하는데, 쪼개 두면 바뀐 부분만 고치면 됩니다.

## 2. 내 세계관을 다섯 문장으로 줄이면

1. 나는 **임상가-빌더**다 — 치료사이면서 직접 도구를 만드는 사람. 순서가 중요하다: 임상가가 먼저다.
2. 만들 것은 치료실에서 **내가 직접 겪은 문제**에서만 나온다. AI가 신기해서 만드는 게 아니다.
3. **한 아이**를 위해 만들어서 통하면, 그때 **많은 아이**용으로 키운다. 순서를 거꾸로 하지 않는다.
4. 잘 만들었는지는 느낌이 아니라 **기록**(아이가 실제로 나아졌는가)으로 판단한다.
5. 아이의 데이터는 아이의 것 — 실명도, 기록도 함부로 내보내지 않는다.

이 다섯 문장이 전부입니다. 나머지 노트는 전부 이 다섯 문장의 **근거·세부 규칙·실행 방법**이에요.

## 3. 폴더가 5개인 이유 — 생각의 조립 순서

| 폴더 | 쉽게 말하면 | 언제 열어보나 |
|---|---|---|
| `10_원문` | **그때의 사진첩.** v0.1, v0.2를 찍어둔 것. 절대 안 고침 | "내 생각이 어떻게 변해왔지?" 궁금할 때 |
| `20_원자노트` | **레고 블록.** 미션·원칙·프로젝트를 낱개로 쪼갠 것. 여기만 고치면 됨 | 평소에 가장 자주 |
| `30_메타분석` | **거울.** 내 문서를 남의 눈으로 뜯어본 기록 | 뭔가 방향이 이상하다 싶을 때 |
| `40_재조합` | **조립 결과.** 거울에서 본 것을 반영해 다시 짠 그림·문서 | 전체 그림이 보고 싶을 때 |
| `50_템플릿` | **체크리스트 양식.** 복사해서 빈칸만 채우면 되는 것 | 새 프로젝트 시작할 때, 3개월마다 |

## 4. 어려운 말 번역표

| 볼트에 나오는 말 | 쉽게 말하면 |
|---|---|
| **초인지 / 메타분석** | 내 생각을 한 발 떨어져서 내려다보기. "나는 왜 이렇게 생각하지?"를 생각하는 것 |
| **원자 노트** | 더 못 쪼갤 만큼 작게 나눈 노트. 블록 하나 = 생각 하나 |
| **스냅샷 / 동결** | 그 시점의 문서를 사진 찍듯 저장하고 다시는 안 건드리는 것. 나중에 "그때 난 이랬구나" 비교용 |
| **논리 레벨** (메타 1) | 사람의 생각을 6층 건물로 보는 틀. 꼭대기가 "왜 사는가", 바닥이 "오늘 뭘 하는가". 진단 결과: 내 건물은 꼭대기는 튼튼한데 3층(능력)·2층(습관)이 비어 있었음 → 그래서 [[능력 재고]]와 [[실행 루틴]]을 만들어 채움 |
| **N=1 → N=다수** | 한 명(N=1)을 위해 만들고, 통하면 여러 명(N=다수)용으로 확장한다는 뜻 |
| **파이프라인 5단계** | 한 아이용이 모두의 도구가 되는 길: ① 치료실에서 문제 발견 → ② 한 아이용으로 만들기 → ③ 효과 확인 → ④ 아이마다 다른 부분을 '설정 메뉴'로 바꾸기 → ⑤ 플랫폼(TalkBloom)에 넣기 |
| **기초선** (효과 기록) | 도구를 쓰기 **전**의 아이 상태를 3번 재 둔 것. 이게 있어야 "도구 덕에 나아졌다"고 말할 수 있음 |
| **코어 트랙 / 탐색 트랙** | 코어 = 언어치료 관련 본업 제품. 탐색 = 그 외(톤메이커 같은 것). 탐색은 놀이가 아니라 **기술 연습장**이고, 배운 기술은 반드시 본업 제품에 다시 씀 |
| **5문 게이트** | 새 프로젝트를 시작하기 전에 통과해야 하는 질문 5개. 하나라도 "아니오"면 시작 안 함 ([[템플릿 — 새 프로젝트 킥오프]]) |
| **위키링크 `[[ ]]`** | 옵시디언에서 노트끼리 연결하는 문법. 클릭하면 그 노트로 이동 |
| **MOC** | Map of Content — 목차 역할 노트. 이 볼트에선 [[00_HOME]] |
| **H 아동 / J 아동** | 실제 아동 두 명의 익명 표기. 공개 저장소라 실명을 쓰지 않음 ([[원칙 6 — 아이의 데이터는 아이의 것이다]]) |

## 5. 메타분석 4개가 각각 알려준 것 (한 줄씩)

- [[메타 1 — 논리 레벨 분석]] → "왜"는 튼튼한데 "어떻게(능력·습관)"가 비어 있었다 → 능력 재고·실행 루틴으로 채움
- [[메타 2 — 언어 패턴 분석]] → 나를 "~가 아니다"로만 설명하고 있었다 → **임상가-빌더**라는 이름을 지음
- [[메타 3 — 긴장과 모순]] → "앱 많이 만들기 싫다"면서 실제론 많이 만들고 있었다 → 탐색 트랙으로 공식화해서 모순을 전략으로 바꿈
- [[메타 4 — 빈칸과 질문]] → 돈·데이터 윤리·효과 증명·동료·경계·중간 목표 6개가 비어 있었다 → 원칙 6·7, 이정표, 거절 목록으로 일부를 채웠고 나머지(돈, 동료)는 알고 비워 둠
