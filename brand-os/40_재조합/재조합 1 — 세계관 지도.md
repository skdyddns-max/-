# 재조합 1 — 세계관 지도

Brand "OS"라는 이름을 진지하게 받아들이면, 세계관은 운영체제처럼 3층으로 조립된다. **아래층일수록 바꾸기 어렵고, 위층일수록 자주 바뀌어야 정상이다.**

```mermaid
flowchart TB
    subgraph APPS["🟢 앱 레이어 — 빠르게 변한다 (제품)"]
        direction TB
        CUR["현재: 맞춤 AAC ×2 · 워크시트 분석기 · 화용 스튜디오"]
        FUT["미래: 치료일지 · 상담 리포트 · 평가 분석기 · AAC 자동 생성 · 자료 플랫폼 · 자동화 · AI 코치"]
        TB["🌸 TalkBloom 통합 플랫폼"]
        EXP["🧪 탐색 트랙: 톤메이커 · 디테일 · 마음터칭<br/>(기술 훈련장 — 배운 것은 코어로 회수)"]
        CUR --> FUT --> TB
    end

    subgraph SYS["🟡 시스템 레이어 — 느리게 변한다 (원칙 = 판단 규칙)"]
        P1["① 시작은 현장에서"]
        P2["② AI는 목적이 아니다"]
        P3["③ 한 아이 → 많은 아이"]
        P4["④ 기술보다 치료 효과"]
        P5["⑤ 직접 만들고·쓰고·개선"]
    end

    subgraph KERNEL["🔴 커널 — 거의 변하지 않는다 (미션·정체성)"]
        M["미션: 치료실의 한계를 넘어<br/>아이에게 정말 필요한 기술을"]
        I["정체성: 현장의 문제를<br/>AI 제품으로 해결하는 사람"]
        V["비전: 언어재활과 AI를 연결해<br/>새로운 가능성을 만든 사람"]
    end

    KERNEL --> SYS --> APPS
    EXP -. 기술 회수 .-> CUR
```

## 가치 전달 사슬 (긴장 3의 해소)

제품 대부분이 치료사용인 이유를 사슬로 명시하면 모순이 사라진다:

```mermaid
flowchart LR
    T["도구"] -->|"문서 시간 ↓"| SLP["치료사"]
    T2["코치·리포트"] -->|"가정 연습의 질 ↑"| P["보호자"]
    SLP -->|"아이와의 시간·집중 ↑"| C["👧 아이"]
    P -->|"치료실 밖 일반화"| C
    AAC["AAC"] -->|"직접"| C
    C -->|"기록·데이터"| V2["효과 검증"]
    V2 -->|"개선"| T
```

**모든 화살표는 결국 아이에서 끝나고, 검증 루프를 타고 도구로 돌아온다.** 이 순환이 [[원칙 5 — 직접 만들고 직접 쓰고 직접 개선한다]]의 확장판이다.

## N=1 → N=다수 파이프라인 (긴장 1의 해소)

```mermaid
flowchart LR
    A["1️⃣ 발견<br/>치료실의 문제"] --> B["2️⃣ 시제품<br/>한 아이 맞춤"] --> C["3️⃣ 검증<br/>효과를 기록으로"] --> D["4️⃣ 일반화<br/>차이점 → 설정 항목"] --> E["5️⃣ 플랫폼<br/>TalkBloom 모듈"]
```

현재 위치: 맞춤 AAC는 2단계 (H·J 두 사례), 워크시트 분석기는 3단계 진입 중, 화용 스튜디오는 4단계(프로젝트 복제·아동별 기록까지 감).
